# Arrays
## Introduction

In software engineering, arrays are one of the most fundamental data structures because they enable us to store and process collections of related data efficiently. Almost every real-world application—from banking and healthcare to insurance, e-commerce, and artificial intelligence—uses arrays to organize information and perform calculations. While one-dimensional arrays help manage lists of similar data, two-dimensional arrays (matrices) are used to represent structured information such as tables, reports, schedules, and business rules.

In our Insurance Management System, we can use matrices to represent premium calculation factors, claim records, branch performance, customer risk assessments, and policy recommendations. Instead of writing long and complex conditional statements, we can store business rules in a matrix and retrieve the appropriate value using row and column indexes. This approach makes the application easier to understand, maintain, and extend as business requirements evolve.

By implementing premium calculation using a matrix, students will discover that arrays are not just an academic topic—they are practical tools used to build scalable and data-driven software solutions. This bridges the gap between programming fundamentals and real-world software engineering.

## Objective

The objective of this exercise is to help students understand how one-dimensional and two-dimensional arrays can solve real business problems through an Insurance Management System. Students will learn how to organize data in arrays, access elements using indexes, perform matrix-based calculations, and integrate these concepts into an object-oriented application. By the end of this project, students will appreciate the importance of arrays as the foundation for advanced data structures, algorithms, data analytics, machine learning, computer graphics, and enterprise software development.

# Chapter 1: What is an Array?

## Definition

An **Array** is a linear data structure that stores **multiple elements of the same data type** in **contiguous memory locations**.

Think of an array as a row of lockers.

```
Index

 0     1     2     3     4

+----+----+----+----+----+
| 10 | 20 | 30 | 40 | 50 |
+----+----+----+----+----+
```

Each locker has

* Address
* Index
* Value

The computer calculates every element's address automatically.

# Key Takeaways

* Arrays store homogeneous data in contiguous memory.
* 1D arrays represent linear collections of elements.
* 2D arrays represent matrices or tabular data.
* Array access by index is very fast (**O(1)**).
* Matrix addition requires identical dimensions.
* Matrix multiplication requires compatible dimensions (columns of the first matrix = rows of the second).
* Arrays are foundational for advanced data structures such as matrices, stacks, queues, heaps, graphs, and dynamic programming, making them an essential topic for software engineering and technical interviews.

## Premium Calculation Matrix into the Insurance Management Application

Until now, our Insurance Management Application has focused on managing customers, policies, and policy issuance using classes, objects, and object-oriented programming principles. While this provides a solid foundation, real-world insurance systems do much more than simply store information. One of their primary responsibilities is calculating insurance premiums accurately based on multiple business factors such as customer age, policy type, occupation, health condition, and risk category.

If we implement these business rules using long `if-else` statements, the application quickly becomes difficult to read, maintain, and extend. Professional software engineers therefore organize such business rules in structured data so that calculations become simpler, faster, and more flexible.

This is where the concept of **two-dimensional arrays (matrices)** becomes valuable. A matrix allows us to represent premium factors in a tabular form, where each row represents a customer category and each column represents a policy type. During premium calculation, the application simply retrieves the appropriate factor from the matrix using row and column indexes instead of evaluating numerous conditional statements.

By introducing a **Premium Calculation Matrix** as a separate service, we also follow an important software engineering principle known as **Separation of Concerns**. The `InsuranceManager` continues to manage customers and policies, while the new `PremiumCalculationService` becomes responsible only for premium computation. This makes the application modular, easier to test, easier to maintain, and ready for future enhancements.

The objective of this enhancement is to integrate a **Premium Calculation Matrix Service** into the existing Insurance Management Application and demonstrate how two-dimensional arrays solve real business problems.

After completing this implementation, students will be able to:

* Understand the practical use of two-dimensional arrays (matrices).
* Store business rules using matrix data instead of complex conditional logic.
* Retrieve matrix values using row and column indexes.
* Calculate insurance premiums based on customer category and policy type.
* Design a dedicated `PremiumCalculationService` that follows good object-oriented design principles.
* Integrate multiple software engineering concepts including Arrays, Classes, Objects, Services, Business Logic, and Modular Design into a single real-world application.

This enhancement demonstrates that arrays are much more than an academic programming topic—they are fundamental building blocks used in enterprise software systems to represent business rules, perform calculations, and build scalable applications.

## Step 5: PremiumCalculator Class

```cpp
class PremiumCalculator
{
private:

    float premiumFactor[3][3] =
    {
        {0.8,0.9,1.0},
        {1.0,1.2,1.1},
        {1.5,1.8,1.4}
    };

    float basePremium[3] =
    {
        5000,
        7000,
        6000
    };

public:

    float CalculatePremium(int customerCategory,
                           int policyType)
    {
        return basePremium[policyType] *
               premiumFactor[customerCategory][policyType];
    }
};
```
