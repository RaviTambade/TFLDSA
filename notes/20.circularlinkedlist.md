# Circular Linked List

Circular Linked Lists are used whenever **the process never really ends** and we need to **keep visiting elements repeatedly**. Instead of reaching `NULL`, we naturally return to the first element.
 
## 1. CPU Round Robin Scheduling 
### Problem

Suppose there are four processes waiting for CPU time.

```
P1 -> P2 -> P3 -> P4
```

Each process gets **2 milliseconds**. After its time slice finishes, the CPU should execute the next process. After P4 finishes... Should the CPU stop? No. It should return to P1.
 

### Circular Linked List Representation

```
      +--------------------+
      |                    |
      v                    |
+-----+    +-----+    +-----+    +-----+
| P1  | -> | P2  | -> | P3  | -> | P4  |
+-----+    +-----+    +-----+    +-----+
   ^                                  |
   |__________________________________|
```

Execution order

```
P1
 ↓
P2
 ↓
P3
 ↓
P4
 ↓
P1
 ↓
P2
 ↓
...
```

### Why Circular?

Because scheduling never ends until all processes finish.

The scheduler simply moves

```c
current = current->next;
```

No special handling for the last process.

#### Why Companies Prefer Circular Lists

Imagine implementing Round Robin using an array.

```
[ P1 P2 P3 P4 ]
```

Every time you reach the end

```c
if(index==n)
    index=0;
```

Extra condition.

With Circular Linked List

```c
current=current->next;
```

When the last node is reached,

its `next` pointer already points to the first node.

No extra logic.

# Mentor's Way of Thinking

Don't memorize **"Circular Linked List is used in Round Robin Scheduling."**

Instead, ask:

> **Does this problem involve visiting items repeatedly in a cycle without a natural end?**

If the answer is **yes**, a Circular Linked List is a strong candidate.

Examples include:

* ✅ CPU processes taking turns
* ✅ Songs repeating in a playlist
* ✅ Players taking turns in a game
* ✅ Traffic lights cycling through directions
* ✅ Machines on an assembly line
* ✅ Network tokens circulating among devices

**Pattern to remember:**

```
Repeated Cycle
      ↓
Need to revisit first element
      ↓
No natural NULL at the end
      ↓
Circular Linked List
```

This way of recognizing the *pattern* is far more valuable in interviews and system design than simply memorizing a list of applications.


## Insurance Processes (Domain)

A **Circular Linked List (CLL)** is useful when business operations need to happen **continuously in a cycle**. Many insurance processes are repetitive, making a circular linked list a natural fit.

## 1. Agent Rotation (Round-Robin Assignment)

Suppose an insurance company has four sales agents:

```
+--------+     +--------+     +--------+     +--------+
| Amit   | --> | Neha   | --> | Rahul  | --> | Priya  |
+--------+     +--------+     +--------+     +--------+
      ^                                              |
      |______________________________________________|
```

When a new customer arrives:

```
Customer 1 → Amit
Customer 2 → Neha
Customer 3 → Rahul
Customer 4 → Priya
Customer 5 → Amit
Customer 6 → Neha
...
```

### Why Circular Linked List?

* Fair distribution
* No agent gets overloaded
* After the last agent, assignment automatically returns to the first.


## 2. Premium Payment Reminder System

Every day the system checks policy renewals.

```
Policy101
    ↓
Policy102
    ↓
Policy103
    ↓
Policy104
    ↓
(back to Policy101)
```

The reminder service continuously traverses the list.

```
while(true)
{
    CheckCurrentPolicy();

    current = current.Next;

    Sleep(1 day);
}
```

No need to restart from the beginning manually.

## Let us try out simple implementation


Below is a  **Circular Singly Linked List** in C# using a Console Application. The goal is to understand **how nodes are connected**, not just to write code.

### Step 1: Create Node Class

```csharp
using System;

namespace CircularLinkedListDemo
{
    class Node
    {
        public int Data;
        public Node Next;

        public Node(int data)
        {
            Data = data;
            Next = null;
        }
    }
}
```

###  Step 2: Create CircularLinkedList Class

```csharp
using System;

namespace CircularLinkedListDemo
{
    class CircularLinkedList
    {
        private Node head;

        public CircularLinkedList()
        {
            head = null;
        }
    }
}
```

 
###  Step 3: Insert at End

```csharp
public void InsertAtEnd(int data)
{
    Node newNode = new Node(data);

    // Empty List
    if (head == null)
    {
        head = newNode;
        newNode.Next = head;
        return;
    }

    Node temp = head;

    while (temp.Next != head)
    {
        temp = temp.Next;
    }

    temp.Next = newNode;
    newNode.Next = head;
}
```

###  Step 4: Display List

```csharp
public void Display()
{
    if (head == null)
    {
        Console.WriteLine("List is Empty");
        return;
    }

    Node temp = head;

    do
    {
        Console.Write(temp.Data + " ");
        temp = temp.Next;

    } while (temp != head);

    Console.WriteLine();
}
```
 

###  Step 5: Search

```csharp
public bool Search(int key)
{
    if (head == null)
        return false;

    Node temp = head;

    do
    {
        if (temp.Data == key)
            return true;

        temp = temp.Next;

    } while (temp != head);

    return false;
}
```



###  Step 6: Insert at Beginning

```csharp
public void InsertAtBeginning(int data)
{
    Node newNode = new Node(data);

    if (head == null)
    {
        head = newNode;
        newNode.Next = head;
        return;
    }

    Node last = head;

    while (last.Next != head)
    {
        last = last.Next;
    }

    newNode.Next = head;
    last.Next = newNode;
    head = newNode;
}
```



###  Step 7: Delete First Node

```csharp
public void DeleteFirst()
{
    if (head == null)
        return;

    if (head.Next == head)
    {
        head = null;
        return;
    }

    Node last = head;

    while (last.Next != head)
    {
        last = last.Next;
    }

    head = head.Next;
    last.Next = head;
}
```

### Step 8: Delete Last Node

```csharp
public void DeleteLast()
{
    if (head == null)
        return;

    if (head.Next == head)
    {
        head = null;
        return;
    }

    Node prev = null;
    Node current = head;

    while (current.Next != head)
    {
        prev = current;
        current = current.Next;
    }

    prev.Next = head;
}
```


###  Step 9: Main Program

```csharp
using System;

namespace CircularLinkedListDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            CircularLinkedList list = new CircularLinkedList();

            list.InsertAtEnd(10);
            list.InsertAtEnd(20);
            list.InsertAtEnd(30);
            list.InsertAtEnd(40);

            Console.WriteLine("Original List");
            list.Display();

            Console.WriteLine();

            Console.WriteLine("Insert 5 at Beginning");
            list.InsertAtBeginning(5);
            list.Display();

            Console.WriteLine();

            Console.WriteLine("Delete First");
            list.DeleteFirst();
            list.Display();

            Console.WriteLine();

            Console.WriteLine("Delete Last");
            list.DeleteLast();
            list.Display();

            Console.WriteLine();

            Console.WriteLine("Searching 20");

            if (list.Search(20))
                Console.WriteLine("Found");
            else
                Console.WriteLine("Not Found");
        }
    }
}
```

###  Memory Representation

After inserting `10`, `20`, `30`, and `40`:

```text
                head
                  |
                  v
      +-----+    +-----+    +-----+    +-----+
      | 10  | -> | 20  | -> | 30  | -> | 40  |
      +-----+    +-----+    +-----+    +-----+
          ^                                  |
          |__________________________________|
```

The key difference from a normal singly linked list is that the **last node (`40`) does not point to `null`; it points back to `head` (`10`)**, creating a continuous loop.

### Mentor's Learning Tip

Don't think of a Circular Linked List as a completely different data structure. It is simply a **Singly Linked List with one changed rule**:

* **Singly Linked List:** `last.Next = null`
* **Circular Linked List:** `last.Next = head`

This one pointer change affects traversal, insertion, and deletion algorithms. Once you understand that relationship, the rest of the implementation becomes much easier.
