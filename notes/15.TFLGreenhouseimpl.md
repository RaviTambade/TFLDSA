## 🌡️ **Greenhouse Climate Example — Object Creation Story**

Below is your **complete C program rewritten in six languages**:
**Java, C++, C#, JavaScript, TypeScript, Python** — each using a 2D greenhouse array, random climate values, and tabular output.

All codes follow the **same logic**:

* Define a `ClimateControl` structure/class

Perfect for teaching **multi-language thinking**.

### 🌟 1. **Java Version**

```java
import java.util.Random;

class ClimateControl {
    float temperature;
    float humidity;
    float lightIntensity;
    float co2;
    float soilMoisture;
}

public class FarmClimate {
    static final int ROWS = 2;
    static final int COLS = 5;

    public static void main(String[] args) {
        ClimateControl[][] farm = new ClimateControl[ROWS][COLS];
        Random rand = new Random();

        // Initialize
        for(int r = 0; r < ROWS; r++) {
            for(int c = 0; c < COLS; c++) {
                farm[r][c] = new ClimateControl();
                farm[r][c].temperature   = rand.nextInt(1500) / 10f;
                farm[r][c].humidity      = rand.nextInt(101);
                farm[r][c].lightIntensity= rand.nextInt(10000);
                farm[r][c].co2           = rand.nextInt(2000) + 300;
                farm[r][c].soilMoisture  = rand.nextInt(101);
            }
        }

        // Display
        System.out.println("Transflower Farm Climate Data (Tambade Mala)");
        System.out.println("-------------------------------------------------");

        for(int r = 0; r < ROWS; r++) {
            for(int c = 0; c < COLS; c++) {
                ClimateControl g = farm[r][c];
                System.out.printf("[R%dC%d] T=%.1f°C H=%.1f%% L=%d Lux CO₂=%.0fppm SM=%.1f%%\t",
                        r+1, c+1, g.temperature, g.humidity, (int)g.lightIntensity, g.co2, g.soilMoisture);
            }
            System.out.println("\n");
        }
    }
}
```

### 🌟 2. **C++ Version**

```cpp
#include <iostream>
#include <iomanip>
#include <random>

struct ClimateControl {
    float temperature;
    float humidity;
    float lightIntensity;
    float co2;
    float soilMoisture;
};

const int ROWS = 2;
const int COLS = 5;

int main() {
    ClimateControl farm[ROWS][COLS];

    std::random_device rd;
    std::mt19937 gen(rd());
    std::uniform_int_distribution<> t(0, 1500);
    std::uniform_int_distribution<> h(0, 100);
    std::uniform_int_distribution<> l(0, 9999);
    std::uniform_int_distribution<> c(300, 2300);
    std::uniform_int_distribution<> s(0, 100);

    for(int r = 0; r < ROWS; r++) {
        for(int x = 0; x < COLS; x++) {
            farm[r][x].temperature   = t(gen) / 10.0f;
            farm[r][x].humidity      = h(gen);
            farm[r][x].lightIntensity= l(gen);
            farm[r][x].co2           = c(gen);
            farm[r][x].soilMoisture  = s(gen);
        }
    }

    std::cout << "Transflower Farm Climate Data (Tambade Mala)\n";
    std::cout << "--------------------------------------------------\n";

    for(int r = 0; r < ROWS; r++) {
        for(int c2 = 0; c2 < COLS; c2++) {
            auto& g = farm[r][c2];
            std::cout << "[R" << r+1 << "C" << c2+1 << "] "
                      << "T=" << g.temperature
                      << "°C H=" << g.humidity
                      << "% L=" << g.lightIntensity
                      << " Lux CO₂=" << g.co2
                      << "ppm SM=" << g.soilMoisture << "%\t";
        }
        std::cout << "\n\n";
    }
}
```

# 🌟 4. **Where Does the Object Live? Heap vs Stack**

Whenever your program runs, OS creates:

- 1️⃣ **Stack** → short-term, for functions and local variables
- 2️⃣ **Heap** → long-term, for objects created with new/malloc

You always say this in class:

> "Thread owns the stack.
> Object lives in the heap."

So when you write:

```java
ClimateControl sensor = new ClimateControl();
```

`sensor` is on **stack**,
actual object is on **heap**.

Same in C++, C#, JS, Python.

# 🌟 5. **Why `this`, `self`, `->`, and `.`?**

Languages use different names but same idea:

| Language | Keyword               | Meaning                     |
| -------- | --------------------- | --------------------------- |
| C++      | `this->x`             | pointer to current object   |
| Java     | `this.x`              | reference to current object |
| C#       | `this.x`              | same as Java                |
| Python   | `self.x`              | reference to instance       |
| JS       | `this.x`              | dynamic binding             |
| C        | uses pointer `obj->x` | manual memory               |

Same concept.
Different way to say “*I am referring to myself*”.

```python
self.soilMoisture = random.uniform(0,100)
```

Each Climate object is one little greenhouse sensor module.

## 🌾 **3. The Farm – Parameterized Constructor**

Now go to your `Farm` class:

```python
def __init__(self, rows, cols):
    self.rows = rows
    self.columns = cols
```

See?
This is a **parameterized constructor**.

If you pass `(2, 5)`:

* 2 rows
* 5 columns
* Total 10 greenhouses

Inside each cell you place one Climate object.

In Java/C++/C# you would write:

```java
new Climate()
```

But Python says:

```python
Climate()
```

No `new` keyword.
Simple, clean, elegant.

## 🧱 **4. Rows & Columns – Nested Loops in Python**

Python version:

```python
for r in range(self.rows):
    row = []
    for c in range(self.columns):
        greenhouse = Climate()
        row.append(greenhouse)
    self.greenhouse.append(row)
```

This is exactly your C language’s double-loop:

```c
for(int r=0; r<ROWS; r++)
    for(int c=0; c<COLS; c++)
```

The logic is same, only the **accent** changes.

## 🔁 **5. Random – Every Language, Different Style**

Yes, Nitish — random plays the same role everywhere, but each language has its own way of expressing it:

* **Java:** `rand.nextFloat()`
* **C#:** `rand.NextDouble()`
* **C++:** `uniform_real_distribution`
* **Python:** `random.uniform(a, b)`
* **JavaScript:** `Math.random()`
* **C:** `rand() % 1500 / 10.0`

Think of it like dialects in India.

Punjabi, Marathi, Tamil, Gujarati —
Different words, same emotions.

Same way:

Different syntax, same logic.
