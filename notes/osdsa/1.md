# Under the surface of Exeuction: Software

Dear curious learner **Welcome to Transflower classroom**. Some people in this classroom are building applications using **C**. Some are using **Java**. Some prefer **C#**.Others are building modern web applications with **JavaScript**. Different languages... but have you ever wondered?

**"What actually happens after you write the code?"**

Let me tell you a story.

### The Story of a Software Factory

Imagine you own a furniture factory.A customer orders a table.You don't immediately hand over a table.First, someone prepares the design.Then workers cut the wood.They assemble the parts.Finally, the finished table reaches the customer. Software development works in exactly the same way.

### Step 1: The Engineer Thinks

Before writing code, a software engineer first thinks about the problem.

> "How should I solve this?"

That thinking becomes **logic**.Logic is nothing but a sequence of steps.

For example:

* Read two numbers.
* Add them.
* Display the result.

Simple.

### Step 2: Logic Becomes Code

Now the engineer converts that logic into programming statements.If you are using **C**, you write a **function**.

```c
int Add(int a, int b)
{
    return a + b;
}
```

If you are using **Java**,

```java
class Calculator
{
    int add(int a, int b)
    {
        return a + b;
    }
}
```

If you are using **C#**,

```csharp
public int Add(int a, int b)
{
    return a + b;
}
```

No matter which language you choose...

The idea is always the same.

**Logic → Programming Statements → Function**

### Step 3: Functions Become Software

One function is useful. Hundreds of functions become a module.Many modules become a library. Many libraries together become an application. That application is what we proudly call **Software.**


### But...

Here comes the interesting question. You wrote the code. Can the CPU understand C? No. Can the processor understand Java? No.
Can the processor understand C#?  No. The CPU understands only one language... **Machine Language (Binary)**
```
10101010
11001010
11110000
```

So somebody has to translate your program. Who does that?



## There Are Two Ways

### 1. Compilation

Languages like

* C
* C++
* Java (partially)
* C#

first convert your source code into another form before execution. Think of a translator.

English Book -> Translated into Marathi


Now people can read it.

Similarly,

Source Code -> Compiler -> Executable Program -> Operating System ->CPU

The compiler performs the translation before execution.


### 2. Interpretation

Languages like JavaScript work differently. Imagine a teacher reading instructions one line at a time.

> Read first line...

Execute. 

> Read second line...

Execute.

> Read third line...

Execute.
 
No complete executable is created first. Execution happens line by line. That process is called **Interpretation.**  For example, Node.js uses the **V8 JavaScript Engine**. The V8 Engine reads JavaScript and executes it.

## Who Actually Executes the Program?

Many students say,

> "C executes."

No.

> "Java executes."

No.

> "C# executes."

No.

Programming languages never execute themselves. The **Operating System** loads your program into memory. The **CPU** fetches one instruction after another. Memory stores the program and data. The operating system manages files, memory, keyboard, mouse, network, and every hardware device. Finally, the CPU performs the actual computation.

## What is Hardware?

Hardware is the physical part of the computer.

Examples include:

* CPU
* Memory (RAM)
* SSD / Hard Disk
* Keyboard
* Mouse
* Monitor
* Network Card

Among all these components, the most important is the **CPU (Central Processing Unit).** It is called the **brain of the computer** because it performs calculations and executes instructions. Today, computers also include another powerful processor called the

**GPU (Graphics Processing Unit).**

Originally designed for graphics, GPUs are now widely used for AI, Machine Learning, scientific computing, and large-scale parallel processing because they can execute thousands of operations simultaneously.

### The Transflower Mentor Thought

As software engineers, we should never think only about programming languages. Languages keep changing. Yesterday it was C. Then came Java. Then C#. Today JavaScript, Python, Go, Rust, and AI-generated code. But one thing never changes:

> **Every software application follows the same journey—human thinking becomes logic, logic becomes code, code is translated into machine instructions, the operating system loads it into memory, and finally the CPU executes it.**

When you understand this journey, you stop thinking like someone who merely writes code. You begin thinking like a **Software Engineer** who understands how computers actually work.
