**"Who actually executes every instruction?"**

## Imagine a Restaurant Kitchen

Suppose you go to a restaurant.You order a Masala Dosa.Who prepares it? Not the waiter. Not the cashier. The **chef** prepares it.

Similarly, The operating system only **brings your program to the CPU.** The actual work is done by the **CPU.**

# CPU is the Brain of the Computer

The CPU performs millions (today even billions) of instructions every second. But how does it do that? It doesn't execute the whole program at once. It executes... **One instruction at a time.** This is called

> **Sequential Execution of Instructions**

Imagine your program contains:

```text
Read Number
‚Üì
Add 10
‚Üì
Multiply by 5
‚Üì
Display Result
```

The CPU executes them one after another.


# Inside the CPU

Many students think the CPU is one single chip doing everything. Actually, inside the CPU there are several important components.

```text
             CPU
     --------------------
     |                  |

     |                  |
```

Each component has a different responsibility.


# Arithmetic Logic Unit (ALU)

Imagine a calculator. What can a calculator do?

* Addition
* Subtraction
* Multiplication
* Division

Similarly, The **Arithmetic Logic Unit (ALU)** performs calculations. Not only arithmetic... It also performs logical operations. For example,

```text
10 + 20
50 - 10
25 √ó 4
A > B
A == B
```

Whenever your program performs calculations,the ALU is responsible.

# But ALU Needs Data

Imagine asking a chef,"Please cook."The chef asks,"Where are the vegetables?"Without ingredients,the chef cannot cook.

Similarly,The ALU asks,
> "Where is my data?"

The data comes from **Registers.**


# What are Registers?

Registers are extremely small memory locations present **inside the CPU itself.** Think of them as the CPU's personal working table. The ALU never goes directly to the hard disk. It never even prefers RAM if it can avoid it.  Instead, it works on data already available inside registers.

```text
RAM
‚Üì
Register
‚Üì
ALU
‚Üì
Result
```

Registers are the fastest memory in the computer.

# Register Size

Registers have fixed sizes. For example,

* 8-bit
* 16-bit
* 32-bit
* 64-bit

If a processor has 64-bit registers, we call it a **64-bit Processor.** Earlier computers had

* 8-bit processors
* 16-bit processors
* 32-bit processors

Today, almost every desktop and laptop uses **64-bit processors.** Larger registers allow the CPU to process larger values and access more memory efficiently.

# RAM (Primary Memory)

RAM stands for **Random Access Memory.** It stores programs that are currently running.
When you start an application, the operating system copies it from the SSD or Hard Disk into RAM. 

The CPU reads instructions from RAM.RAM is much faster than secondary storage but loses its contents when power is turned off.


# Hard Disk / SSD (Secondary Storage)

This is permanent storage. It stores:

* Windows
* Linux
* Applications
* Photos
* Videos
* Documents

Even after the computer is switched off, the data remains safe.


# Cache Memory

Cache is a very fast memory located inside or very close to the CPU.Modern processors usually contain multiple levels of cache:

* L1 Cache ‚Äì smallest and fastest
* L2 Cache ‚Äì larger and slightly slower
* L3 Cache ‚Äì shared among cores and larger still

The CPU checks the cache before accessing RAM because cache access is much faster.

# The Transflower Mentor Thought

Many beginners believe that learning programming means learning only **C**, **Java**, or **C#** syntax.A software engineer thinks differently.When you write:

```c

```
you should be able to imagine the invisible journey:

* The operating system loads the program into RAM.
* The CPU fetches the instruction.
* Required data moves through cache into registers.
* The ALU performs the multiplication.
* The result is stored back in a register and then in memory.
