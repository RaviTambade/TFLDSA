# Linked List

As our Insurance Management Application grows, we quickly realize that not all data can be managed efficiently using arrays. Arrays require a fixed size, and adding or removing customers, policies, or claims often involves shifting elements, resulting in unnecessary processing and memory usage. In a real insurance company, new customers register every day, policies are issued and renewed, claims are filed, and records are updated continuously. Since the amount of data is dynamic, we need a more flexible data structure.

This is where the **Linked List** becomes an essential data structure. Unlike an array, a linked list does not require contiguous memory locations or a fixed size. Each record is stored in a separate node, and every node maintains a link (pointer) to the next node. This allows the application to grow dynamically by allocating memory only when new records are added.

In our Insurance Management Application, a linked list can be used to manage customers, insurance policies, claim records, premium payment history, branch offices, and support tickets. Whenever a new customer purchases a policy, a new node is created and linked to the existing list. Similarly, when a policy expires or a customer closes an account, the corresponding node can be removed without reorganizing the remaining records.

By integrating the **Linked List** into our existing application, we replace the limitations of fixed-size arrays with a dynamic and scalable data structure. This helps students understand how professional software systems manage continuously changing business data while applying both programming fundamentals and object-oriented design principles.

The objective of this enhancement is to introduce the **Linked List Data Structure** by extending the existing Insurance Management Application. Students will learn how to represent insurance records as linked nodes, dynamically allocate memory, traverse records, search for specific policies or customers, insert new records, delete obsolete records, and update existing information efficiently.

After completing this implementation, students will be able to:

* Understand the limitations of arrays for dynamic business data.
* Learn the structure of a linked list node consisting of data and a pointer.
* Dynamically create and manage insurance records using linked lists.
* Perform insertion, deletion, searching, traversal, and update operations.
* Integrate linked lists into an object-oriented Insurance Management System.
* Appreciate why linked lists are widely used in enterprise software for managing continuously changing datasets.

This enhancement demonstrates that while arrays are excellent for fixed-size collections and matrix-based calculations such as premium computation, **linked lists are the preferred choice when business data grows and changes dynamically**, making them a fundamental building block for scalable software systems.


#### 🔥 **Array vs. Linked List — Complete Comparison**


#### **Array**

* A **fixed-size** collection of elements stored in **contiguous memory**.
* All elements are stored **next to each other**.
* Example: `int arr[5] = {10,20,30,40,50};`

#### **Linked List**

* A **dynamic-size** collection of nodes stored at **non-contiguous memory**.
* Each node contains:

  * **data**
  * **pointer** to the next node
* Example node:
  `| data | next | →`

---

## 2️⃣ **Memory Layout**

#### **Array Memory (Contiguous)**

```
Index:   0    1    2    3    4
        +----+----+----+----+----+
Data:   | 10 | 20 | 30 | 40 | 50 |
        +----+----+----+----+----+
```

#### **Linked List Memory (Non-Contiguous)**

```
+-------+    +-------+    +-------+
|  10   | -> |  20   | -> |  30   | -> NULL
+-------+    +-------+    +-------+
```

Each block can be anywhere in memory.

## 3️⃣ **Key Differences**

| Feature                | Array                          | Linked List                                     |
| ---------------------- | ------------------------------ | ----------------------------------------------- |
| **Memory allocation**  | Static / fixed                 | Dynamic / grows as needed                       |
| **Memory location**    | Contiguous                     | Random / non-contiguous                         |
| **Size**               | Fixed at creation              | Flexible, can grow/shrink                       |
| **Access time**        | O(1) random access using index | O(n) sequential access                          |
| **Insertion/Deletion** | Slow: O(n) (shifting required) | Fast: O(1) at beginning/middle if pointer known |
| **Extra memory**       | No extra memory                | Extra memory for pointers                       |
| **Cache friendliness** | Very good (contiguous)         | Poor                                            |
| **Implementation**     | Simple                         | Requires understanding of pointers              |
| **Best used when**     | Frequent access by index       | Frequent insert/delete                          |


## 4️⃣ **Advantages & Disadvantages**

#### **Array – Advantages**

* Fast random access (`arr[i]`)
* Good cache performance
* Simple to use

#### **Array – Disadvantages**

* Fixed size
* Costly insertions and deletions
* Memory waste if array is too large

 

#### **Linked List – Advantages**

* Dynamic size (grows/shrinks at runtime)
* Easy insertion/deletion
* No memory wastage

#### **Linked List – Disadvantages**

* No random access (must traverse)
* Extra memory for pointers
* Poor cache locality
* Slower traversal compared to arrays



## 5️⃣ **When to use What?**

#### Use **Array When:**

* You need **fast access** by index.
* Size is **fixed or known in advance**.
* Data is used frequently (good cache performance).

#### Use **Linked List When:**

* You need **frequent insertions or deletions**.
* You don’t know size beforehand.
* Memory fragmentation is acceptable.

# 🎯 Short and Sweet Exam-Ready Answer

> **Array** is a *contiguous* block of memory with *fixed size* and *O(1) random access*, but insert/delete operations are slow.
> **Linked list** stores elements in *non-contiguous nodes* connected by pointers, supports *dynamic size* and *fast insert/delete*, but does not support random access and uses extra memory for pointers.

## 🧩 **STEP 1 — Imagine the Linked List as a Train**

Before writing even one line of code, imagine a **train of bogies**.

* Every bogie = one **Node**
* The coupling between bogies = the **next pointer**
* The first bogie = the **head pointer**
* The last bogie’s next = **NULL** (end of train)

So what should one **Node** contain?

1. Data
2. Pointer to the next node

Exactly like this:

```
+--------+--------+
| data   |  next  | ---> next bogie
+--------+--------+
```

So before writing big logic, we define a *tiny* building block — the **Node**.


## 🧩 **STEP 2 — Write node.h like an Engineer Sketching a Blueprint**

```cpp
class Node {
public:
    int data;
    Node* next;

    Node(int value);
};
```

Mentor:
"See? This is pure thinking turned into a minimal blueprint.
Data + Pointer to same type → Self-referential structure → Backbone of linked list."

## 🧩 **STEP 3 — node.cpp: Giving Life to Each Bogie**

```cpp
Node::Node(int value)
    : data(value), next(nullptr) {}
```

Mentor:
"When a bogie is created, it gets its number (`data`)
and its connector (`next`) points to `nullptr` because it is free — not linked yet."

## 🧩 **STEP 4 — Create the LinkedList Class Skeleton (Reverse Engineering Style)**

Before writing any logic, define an empty structure:

```cpp
class LinkedList {
private:

public:
    LinkedList();

};
```

Mentor:
"Notice something important:
I have *declared* all functions but written **0 lines of logic** yet.
This is your programmer’s sketch — your 'Cortana' on paper."


## 🧩 **STEP 5 — linkedlist.cpp (Step-by-step Logic Filling)**

### Step 5.1 — Start with Constructor

```cpp
LinkedList::LinkedList()
    : head(nullptr) {}
```

Mentor:
"Every linked list starts EMPTY. Head = NULL."

### Step 5.2 — **Insert Logic (Append): Two Scenarios**

#### ✦ Scenario 1: List Is Empty

```cpp
if (head == nullptr) {
    head = newNode;
    return;
}
```

“First bogie becomes the head.”

#### ✦ Scenario 2: List Is Not Empty

Traverse to end and attach bogie:

```cpp
Node* curr = head;
while (curr->next != nullptr) {
    curr = curr->next;
}
curr->next = newNode;
```

Mentor analogy:

> “Current = bogie. Keep walking till you reach the last bogie whose next is NULL. Then attach the new bogie.”

### Step 5.3 — **Display Logic: Traversing the Train**

```cpp
Node* curr = head;

if (!curr) {
    std::cout << "(empty)\n";
    return;
}

while (curr != nullptr) {
    std::cout << curr->data;
    if (curr->next) std::cout << " -> ";
    curr = curr->next;
}
std::cout << " -> NULL\n";
```

Mentor:

> “Start at head. Walk bogie by bogie until you hit NULL.”

### Step 5.4 — **Remove Logic (first occurrence)**

You remove a bogie in 3 cases:

1. List empty
2. Value is in head
3. Value is somewhere in between

Implementation:

```cpp
bool LinkedList::remove(int value) {

    if (!head) return false;

    if (head->data == value) {
        Node* temp = head;
        head = head->next;
        delete temp;
        return true;
    }

    Node* curr = head;
    while (curr->next && curr->next->data != value) {
        curr = curr->next;
    }

    if (!curr->next) return false;

    Node* temp = curr->next;
    curr->next = temp->next;
    delete temp;
    return true;
}
```

Mentor:
“Careful!
Removing from middle requires reconnecting bogies properly.”


### Step 5.5 — **Search Logic**

```cpp
bool LinkedList::search(int value) const {
    Node* curr = head;
    while (curr) {
        if (curr->data == value) return true;
        curr = curr->next;
    }
    return false;
}
```

### Step 5.6 — **Clear Logic**

```cpp
void LinkedList::clear() {
    Node* curr = head;
    while (curr) {
        Node* temp = curr;
        curr = curr->next;
        delete temp;
    }
    head = nullptr;
}
```

Mentor:
"This is cleaning the entire train — removing all bogies safely."

# Chapter 1: Why Do We Need Linked Lists?

Imagine you are developing software for an insurance company.

Every day,

* New customers register.
* New insurance policies are issued.
* Existing customers renew policies.
* Some policies expire.
* Some customers cancel their policies.

Initially, you stored customers in an array.

```
Customer customers[100];
```

This works only until 100 customers are reached.

Now suppose the 101st customer arrives.

What should we do?

* Increase array size?
* Copy all data into a larger array?
* Waste memory by allocating a huge array?

Arrays have a **fixed size**, making them unsuitable for continuously growing business data.

We need a data structure that grows only when needed.

This is the purpose of a **Linked List**.

# Learning Outcome

By implementing these operations in the Insurance Management System, students learn how linked lists solve real-world problems involving dynamic data. They understand why arrays are suitable for fixed-size collections, while linked lists are ideal for applications where records are frequently added, searched, updated, or removed. This sequence is designed to be implemented incrementally in C or C++ or C# or Java or Python, with each chapter corresponding to one coding exercise that extends the same Insurance Management application rather than introducing isolated examples.


Below is a **simple, beginner-friendly C# Console Application** that builds the linked list step by step using the **Insurance Management System**. The code intentionally avoids advanced C# features so students can focus on understanding **Linked Lists**.

```csharp
using System;

namespace InsuranceManagementSystem
{
    // Represents one insurance customer
    class CustomerNode
    {
        public int CustomerId;
        public string CustomerName;
        public string PolicyType;
        public double Premium;

        // Pointer to next customer
        public CustomerNode Next;
    }

    class InsuranceLinkedList
    {
        private CustomerNode head = null;

        // Create/Add Customer
        public void AddCustomer(int id, string name, string policy, double premium)
        {
            CustomerNode newNode = new CustomerNode();

            newNode.CustomerId = id;
            newNode.CustomerName = name;
            newNode.PolicyType = policy;
            newNode.Premium = premium;
            newNode.Next = null;

            if (head == null)
            {
                head = newNode;
            }
            else
            {
                CustomerNode current = head;

                while (current.Next != null)
                {
                    current = current.Next;
                }

                current.Next = newNode;
            }

            Console.WriteLine("Customer Added Successfully.");
        }

        // Display Customers
        public void DisplayCustomers()
        {
            if (head == null)
            {
                Console.WriteLine("No Customers Found.");
                return;
            }

            CustomerNode current = head;

            Console.WriteLine("-------------------------------------------");

            while (current != null)
            {
                Console.WriteLine("Customer Id : " + current.CustomerId);
                Console.WriteLine("Name        : " + current.CustomerName);
                Console.WriteLine("Policy      : " + current.PolicyType);
                Console.WriteLine("Premium     : " + current.Premium);
                Console.WriteLine("-------------------------------------------");

                current = current.Next;
            }
        }

        // Search Customer
        public void SearchCustomer(int id)
        {
            CustomerNode current = head;

            while (current != null)
            {
                if (current.CustomerId == id)
                {
                    Console.WriteLine("Customer Found");
                    Console.WriteLine(current.CustomerName);
                    return;
                }

                current = current.Next;
            }

            Console.WriteLine("Customer Not Found.");
        }

        // Insert at Beginning
        public void InsertAtBeginning(int id, string name, string policy, double premium)
        {
            CustomerNode newNode = new CustomerNode();

            newNode.CustomerId = id;
            newNode.CustomerName = name;
            newNode.PolicyType = policy;
            newNode.Premium = premium;

            newNode.Next = head;
            head = newNode;

            Console.WriteLine("Customer Inserted At Beginning.");
        }

        // Delete First Customer
        public void DeleteFirst()
        {
            if (head == null)
            {
                Console.WriteLine("List Empty.");
                return;
            }

            head = head.Next;

            Console.WriteLine("First Customer Deleted.");
        }

        // Delete by Customer Id
        public void DeleteCustomer(int id)
        {
            if (head == null)
            {
                Console.WriteLine("List Empty.");
                return;
            }

            if (head.CustomerId == id)
            {
                head = head.Next;
                Console.WriteLine("Customer Deleted.");
                return;
            }

            CustomerNode current = head;

            while (current.Next != null &&
                   current.Next.CustomerId != id)
            {
                current = current.Next;
            }

            if (current.Next == null)
            {
                Console.WriteLine("Customer Not Found.");
            }
            else
            {
                current.Next = current.Next.Next;
                Console.WriteLine("Customer Deleted.");
            }
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            InsuranceLinkedList insurance = new InsuranceLinkedList();

            // Add Customers
            insurance.AddCustomer(101, "Rahul", "Health", 8500);
            insurance.AddCustomer(102, "Priya", "Vehicle", 6200);
            insurance.AddCustomer(103, "Amit", "Life", 9500);

            Console.WriteLine("\nInitial Customer List");
            insurance.DisplayCustomers();

            // Search
            Console.WriteLine("\nSearching Customer...");
            insurance.SearchCustomer(102);

            // Insert at Beginning
            Console.WriteLine("\nAdding VIP Customer...");
            insurance.InsertAtBeginning(100, "Anil", "Life", 12000);

            insurance.DisplayCustomers();

            // Delete First
            Console.WriteLine("\nDeleting First Customer...");
            insurance.DeleteFirst();

            insurance.DisplayCustomers();

            // Delete Customer
            Console.WriteLine("\nDeleting Customer 102...");
            insurance.DeleteCustomer(102);

            insurance.DisplayCustomers();

            Console.ReadKey();
        }
    }
}
```
