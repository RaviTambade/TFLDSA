Memory layout is one of the best ways to help students understand **how data structures actually occupy memory**. Below are simple diagrams you can draw on a whiteboard.

# 1. Structure (Customer)

```text
Memory

Address      Data
----------------------------------------
1000         CustomerID = 101
1004         Name = "Ravi"
1024         Age = 25
1028         Mobile = "9876543210"
1044         Address = "Pune"
----------------------------------------
```

Think of a structure as **one box containing related information**.

```
+----------------------------------+
| Customer                         |
|----------------------------------|
| CustomerId : 101                 |
| Name       : Ravi                |
| Age        : 25                  |
| Mobile     : 9876543210          |
| Address    : Pune                |
+----------------------------------+
```

# 9. Max Heap (Highest Premium)

```

```
Index

A heap is a **tree conceptually**, but it is stored as an **array** in memory.

## Final Teaching Sequence

For every new topic, draw these five views:

1. **Business View** – What problem are we solving?
2. **Logical View** – How is the data organized?
3. **Memory View** – Where is the data stored in memory?
4. **Code View** – How do we implement it?
5. **Application View** – How is it used in the Insurance Management System?

This progression helps students connect business requirements to data structures, memory organization, implementation, and real-world software design.
