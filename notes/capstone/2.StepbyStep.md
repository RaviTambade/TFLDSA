## Teaching Strategy

For every increment, follow the same mentor-driven cycle:

1. **Business Problem** – Why does this feature matter?
2. **Choose the Right Data Structure** – Why is it suitable?
3. **Draw the Memory Layout** – Visualize how data is stored.
4. **Implement Core Operations** – Insert, search, update, delete, traverse.
5. **Integrate with the Existing System** – Reuse previous modules.
6. **Test with Real Insurance Data** – Make it behave like a production application.

This approach helps students experience how **real enterprise software is built incrementally**, with each new business requirement leading naturally to the adoption of an appropriate data structure.
