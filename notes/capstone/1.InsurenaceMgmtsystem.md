# Capstone Project: Insurance Management System

*A complete project to teach Data Structures through a real-world application*

The idea is simple: instead of teaching data structures one by one, students build **one enterprise application** where each feature naturally requires a different data structure.

---

# Project Vision

Imagine you are developing software for an insurance company.

The system must manage:

* Customers
* Insurance Policies
* Claims
* Premium Payments
* Branch Offices
* Employees
* Policy Categories
* Login System
* Reports

Students gradually build this application while learning data structures.

---

# Phase 1 — Structures (Modeling Business Data)

Create business entities.

```c
struct Customer
{
    int customerId;
    char name[50];
    char mobile[15];
    char city[30];
};

struct Policy
{
    int policyId;
    int customerId;
    float premium;
    int termYears;
};
```

Students learn:

* Structure
* Nested structures
* Arrays of structures

# Phase 4 — Linked List

Insurance claims arrive daily.

Claims keep increasing.

Perfect use case.

```
Claim1
   ↓
Claim2
   ↓
Claim3
   ↓
NULL
```

Operations

* Add Claim
* Delete Claim
* Search Claim
* Traverse

Students understand why linked lists are better than fixed arrays for dynamic data.

## Learning Outcome

By the end of this capstone project, students won't just know individual data structures—they will understand **why each one exists, where it fits in enterprise software, and how multiple data structures work together to build a complete Insurance Management System**. This mirrors how real software engineers design applications, choosing the right data structure for each business requirement rather than forcing one solution everywhere.
