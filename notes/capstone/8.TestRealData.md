# Final System Integration Test

After completing all increments, run an end-to-end scenario:

1. Register **Customer C101 – Ravi Tambade**.
2. Create **Policy P1001 (Health Insurance)**.
3. Calculate premium using the **Premium Matrix**.
4. Store the policy in the **BST**.
5. Register **Claim CL101** using the **Linked List**.
6. Add the claim to the **Queue** for processing.
7. If it is an emergency, move it to the **Priority Queue**.
8. Update the customer's mobile number, then use the **Stack** to undo the change.
9. Log in using the **Hash Table**.
10. Generate a report showing the **highest premium** using the **Heap**.
11. Find the nearest inspection route using the **Graph**.

## Mentor's Teaching Tip

For every feature, encourage students to answer these questions before coding:

* **What business problem am I solving?**
* **What real insurance data will I use for testing?**
* **What should the expected output be?**
* **Does the output match the business requirement?**

This habit helps students think like professional software engineers, where testing validates that the software solves real business problems rather than simply compiling successfully.
